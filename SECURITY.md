# Security Policy

Report suspected vulnerabilities **privately** by emailing security@wokey.ai. Please do not open
public issues for security-sensitive reports.

Especially relevant to this project: weaknesses in attestation verification, the signing
statement / proof format, the reproducible build (PCR0), upstream certificate validation, or
anything that could let a tampered or forged response pass verification.

Supported: the latest published release and the immediately previous one.
